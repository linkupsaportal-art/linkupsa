اسم المشروع:
Digital Product Delivery Platform

وصف المشروع:
أحتاج تطوير منصة خاصة لتسليم المنتجات الرقمية بشكل تلقائي واحترافي، مستقلة بالكامل، مع لوحة تحكم كاملة

ملاحظة مهمة:
المشروع يجب أن يكون مملوك لي بالكامل بعد التسليم، مع تسليم السورس كود وقاعدة البيانات وجميع الصلاحيات وملفات الإعداد، بحيث أستطيع التعديل والتطوير عليه مستقبلاً .

طلب 11 → حساب 1

========================================
ربط المتجر المستقل Storefront Integration

الربط مع بوابة المتجر أو الدفع أساسي من البداية.

المطلوب:

- ربط Storefront API
- ربط Webhooks
- استقبال الطلبات تلقائياً
- جلب بيانات الطلب تلقائياً
- جلب رقم الطلب
- جلب اسم العميل
- جلب رقم الجوال
- جلب البريد الإلكتروني
- جلب المنتج
- جلب خيار المنتج
- التحقق من حالة الدفع

تنفيذ الطلب فقط إذا كان:

- paid
- completed

عدم تنفيذ الطلب إذا كان:

- pending
- refunded
- cancelled
- failed

طريقة العمل:

عند إنشاء طلب جديد في المتجر:
↓
يصل Webhook للنظام
↓
النظام يجلب تفاصيل الطلب من Storefront API أو قاعدة البيانات
↓
يتأكد من حالة الدفع
↓
يتأكد من المنتج وخيار المنتج
↓
يتأكد هل رقم الجوال ممنوع من المنتج أم لا
↓
يختار الحساب أو الكود المناسب
↓
يربط الطلب بالحساب
↓
يرسل إشعار للعميل
↓
العميل يدخل رابط الاستلام ويضع رقم الطلب
↓
تظهر له بيانات المنتج

إذا دخل العميل رقم الطلب ولم يكن محفوظاً:
يقوم النظام بجلبه مباشرة من Storefront API ثم تنفيذه إذا كان صحيحاً ومدفوعاً.

========================================
التقنيات المعتمدة (Stack النهائي)

> ✅ تم الاتفاق عليها في محادثة 27/05/2026 — بدون VPS، كل شيء Serverless / Edge.

Frontend:

- Next.js 15 (App Router, Server Components, Server Actions)
- TailwindCSS 4
- shadcn/ui (مكتبة الـ UI الرئيسية — clean modern aesthetic)
- next-intl للدعم الكامل للعربية + RTL
- React Hook Form + Zod للنماذج والـ validation
- TanStack Query لإدارة data fetching على client

Backend / Compute:

- Supabase Edge Functions (Deno) — منطق الأعمال، توزيع الحسابات، توليد TOTP
- Supabase Database Functions (PL/pgSQL) — Round Robin atomic مع SELECT FOR UPDATE SKIP LOCKED
- Supabase Cron (pg_cron) — أرشفة، تنظيف OTP Logs، نسخ احتياطي يومي
- Cloudflare Workers — استقبال Webhooks، Rate Limiting، Bot Protection على edge
- Cloudflare KV — caching للجلسات والـ rate limiters
- Cloudflare Queues — طابور إشعارات (WhatsApp/SMS/Email/Telegram)
- نظام Queue/Workers مخصص (custom) — مبني فوق Supabase + Cloudflare بدون خدمات خارجية

Database:

- Supabase PostgreSQL 15+
- Row Level Security (RLS) للصلاحيات على مستوى الصف
- pgsodium / vault لتشفير الحقول الحساسة (passwords, 2FA secrets, Steam shared_secret)
- pg_cron للجدولة الداخلية

Auth:

- Supabase Auth (Admin Panel + Code-Limit Panel — منفصلين بـ RBAC)
- TOTP 2FA إجباري على الـ Admin
- Customer flow بدون auth — يعتمد على (رقم الطلب + آخر 4 من الجوال) + session مؤقتة

Storage:

- Supabase Storage — لتخزين الملفات الرقمية (Digital Files) مع signed URLs محدودة الصلاحية

Email:

- Resend — للإيميلات الترانساكشنال (إشعار الطلب، استرداد، إلخ)
- React Email للقوالب

Hosting / Deployment:

- Vercel — استضافة Next.js (Frontend + لوحات التحكم) + SSL تلقائي
- Cloudflare — DNS + WAF + DDoS Protection + Turnstile (Captcha مجاني)
- GitHub — Source control + CI/CD مرتبط بـ Vercel

Notifications Channels:

- Email عبر Resend
- WhatsApp عبر Meta Cloud API الرسمي (يحتاج تأكيد العميل)
- SMS عبر مزود محلي سعودي (Unifonic / Mobily — يحتاج تأكيد العميل)
- Telegram Bot API الرسمي

Storefront Integration:

- ربط مباشر عبر API مستقل وبوابات الدفع
- Webhooks مؤمنة مع HMAC signature verification
- Cloudflare Worker كـ webhook receiver أمام Supabase (cold-start ~10ms)

Observability:

- Supabase Logs + Analytics
- Cloudflare Analytics + Logpush
- Sentry للـ error tracking على client + edge functions

Cost Profile:

- صفر VPS — بدون تكاليف ثابتة شهرية للسيرفر
- Vercel Hobby/Pro حسب الحاجة
- Supabase Free → Pro حسب الاستخدام
- Cloudflare Free يكفي للبداية
- التكلفة الفعلية تنمو مع الاستخدام فقط

Ownership Guarantee (التزام صريح):

- السورس كود كامل على GitHub repo خاص بالعميل
- قاعدة البيانات + Schema migrations + RLS policies كلها داخل الـ repo
- جميع credentials و config في حساب العميل (Vercel/Supabase/Cloudflare/GitHub/Resend)
- النظام قابل للنقل لأي Postgres + Node host في المستقبل بدون vendor lock-in قاتل
